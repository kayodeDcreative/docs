This guide will help you integrate Gately authentication into your application in just a few minutes.

## Step 1: Get Your Project ID

First, you'll need to create a Gately project and get your project ID:

1. Sign up at [dashboard.gately.com](https://dashboard.gately.com)
2. Create a new project
3. Copy your project ID from the dashboard

## Step 2: Install the SDK


## Step 3: Initialize the Client

```javascript
import { GatelyBrowserClient } from '@gately/sdk'

const gately = new GatelyBrowserClient('your-project-id')
```

## Step 4: Add Authentication

### Basic Login Form

```html
<form id="login-form">
  <input type="email" name="email" placeholder="Email" required>
  <input type="password" name="password" placeholder="Password" required>
  <button type="submit">Login</button>
</form>

<div id="user-info" style="display: none;">
  <p>Welcome, <span id="user-email"></span>!</p>
  <button id="logout-btn">Logout</button>
</div>
```

### JavaScript Logic

```javascript
// Handle login form
document.getElementById('login-form').addEventListener('submit', async (e) => {
  e.preventDefault()
  const formData = new FormData(e.target)
  
  try {
    await gately.login(
      formData.get('email'),
      formData.get('password')
    )
  } catch (error) {
    alert('Login failed: ' + error.message)
  }
})

// Handle logout
document.getElementById('logout-btn').addEventListener('click', async () => {
  await gately.logout()
})

// Listen for auth state changes
gately.onAuthStateChange((user) => {
  if (user) {
    // User is logged in
    document.getElementById('login-form').style.display = 'none'
    document.getElementById('user-info').style.display = 'block'
    document.getElementById('user-email').textContent = user.email
  } else {
    // User is logged out
    document.getElementById('login-form').style.display = 'block'
    document.getElementById('user-info').style.display = 'none'
  }
})
```

## Step 5: Add Sign Up (Optional)

```javascript
// Add signup functionality
async function signup(email, password) {
  try {
    await gately.signup(email, password, {
      tier: 'basic',
      role: 'user'
    })
    console.log('Signup successful!')
  } catch (error) {
    console.error('Signup failed:', error.message)
  }
}
```

## React Quick Start

If you're using React, here's a complete component:

```jsx
import React from 'react'
import { useGately } from '@gately/sdk'

function App() {
  const { user, login, logout, signup, loading, error } = useGately('your-project-id')
  const [email, setEmail] = React.useState('')
  const [password, setPassword] = React.useState('')

  const handleLogin = async (e) => {
    e.preventDefault()
    try {
      await login(email, password)
    } catch (error) {
      console.error('Login failed:', error)
    }
  }

  if (loading) return <div>Loading...</div>

  if (user) {
    return (
      <div>
        <h1>Welcome, {user.email}!</h1>
        <button onClick={logout}>Logout</button>
      </div>
    )
  }

  return (
    <form onSubmit={handleLogin}>
      <input
        type="email"
        placeholder="Email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        required
      />
      <input
        type="password"
        placeholder="Password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        required
      />
      <button type="submit">Login</button>
      {error && <p style={{color: 'red'}}>{error}</p>}
    </form>
  )
}

export default App
```

## Testing Your Integration

1. **Sign Up**: Create a test user account
2. **Sign In**: Log in with your test credentials
3. **Check State**: Verify the user object is populated
4. **Sign Out**: Test the logout functionality

```javascript
// Test authentication state
console.log('Is authenticated:', gately.isAuthenticated())
console.log('Current user:', gately.getUser())
console.log('Current session:', gately.getSession())
```

## Common Issues

<AccordionGroup>
  <Accordion title="Authentication not working">
    - Check your project ID is correct
    - Verify your domain is whitelisted in the dashboard
    - Check browser console for errors
  </Accordion>
  <Accordion title="User state not persisting">
    - Ensure cookies are enabled
    - Check if you're in incognito/private mode
    - Verify localStorage is available
  </Accordion>
  <Accordion title="CORS errors">
    - Add your domain to allowed origins in the dashboard
    - Check if you're using the correct API URL
    - Ensure HTTPS in production
  </Accordion>
</AccordionGroup>
